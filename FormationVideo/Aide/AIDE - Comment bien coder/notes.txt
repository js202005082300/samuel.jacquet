Comment bien coder

Avoir les bons outils (Autocomplétions, éditeur de code à coloration syntaxique, compilateur selon le langage pour créer un exécutable, …) ;
Travailler en UTF-8 (encodage de gestion de tous les caractères) ;
Se conformer à des conventions (et s’y tenir) 
1)	Travailler avec des tabulations “en 4/8 colonnes” ou des espaces ;
2)	Nomage des variables :
a)	PascalCase = MaVariable (ex : langage Pascal, objet en PHP)
b)	camelCase = maVariable (ex : langage C, procédure en PHP)
c)	snake_case = ma_variable (ex : langage Python)
3)	Longueur des noms des variables (Le nom des variables doit être suffisant pour comprendre sans commentaire : “listeEtudiantAdmisAuBac” ou encore “Int tabEtudiantsAdmisBac[] ;”
4)	Faire confiance à son code (ne pas attendre de nouvelle version de librairies pour sortir son code) ;
5)	Utiliser la documentation (manuel PHP, C++ référence, <language> doc, javascript doc, javascript manual reference) ;
6)	Coder, coder et coder encore !
